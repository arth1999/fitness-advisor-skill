# 第2章 — SFMA 选择性功能动作评估与分解算法

## SFMA 系统定位

选择性功能动作评估（Selective Functional Movement Assessment, SFMA）是面向临床医生的诊断性评估工具，与FMS的角色有本质区别：

| 维度 | FMS | SFMA |
|------|-----|------|
| 目标人群 | 无痛/无症状人群 | 有疼痛或功能障碍的人群 |
| 使用角色 | 教练、运动损伤防护师、临床医生 | 临床医生（整脊师、物理治疗师等） |
| 目的 | 识别风险、建立功能基线 | 诊断功能障碍根源、指导治疗 |
| 场景 | 赛前筛查、训练前评估 | 临床初诊、疼痛管理、术后康复 |
| 动作层级 | 全面健康层 | 身体健康层 |

> "FMS和SFMA的第一个目标是发现功能模式中的问题。在测试中，在患者或客户做每一个动作前你说的最后一句话应该是：'如果你在做这个动作时感到任何疼痛，请告诉我。'"

---

## SFMA 四象限诊断系统

SFMA将每个动作模式归类到四个象限中，分离了疼痛行为与功能障碍行为的语言和视觉：

```
        无疼痛(Non-painful)        有疼痛(Painful)
         ┌────────────────┬────────────────┐
功能正常 │      FN        │      FP        │ → 保护
(Functional) │  可安全发展     │  需保护         │
         ├────────────────┼────────────────┤
功能障碍 │      DN        │      DP        │ → 保护+纠正
(Dysfunctional)│  需要纠正      │  需保护+纠正    │
         └────────────────┴────────────────┘
```

### 四象限详解

| 分类 | 名称 | 表现 | 行动 |
|------|------|------|------|
| **FN** | 功能正常、无疼痛 | 完全无限制的动作，无疼痛 | 可安全发展 |
| **FP** | 功能正常、有疼痛 | 完全无限制的动作，伴疼痛 | 保护——避免直接挑战，需深入调查 |
| **DN** | 功能障碍、无疼痛 | 有限的、受限的动作，无疼痛 | 纠正——这是最有价值的干预目标 |
| **DP** | 功能障碍、有疼痛 | 有限的、受限的动作，伴疼痛 | 保护+纠正——先处理疼痛再解决功能障碍 |

**关键洞察：**
- 疼痛的模式（FP或DP）中有功能障碍也可能没有——如果不明确区分，可能把疼痛模式误当作需要"纠正练习"的"功能障碍模式"
- "对于有疼痛感的关节或动作模式，在此基础上加入纠正练习就像用手指猛击车门，希望自己的手感觉好些"
- DN模式是纠正策略的最佳切入点——可以在不触发疼痛的情况下安全地改善功能

## SFMA 首要层级动作

SFMA检查7个首要层级（Top Tier）动作，按复杂度从简单到高级排列：

```
颈椎 → 肩部 → 多部位屈曲或伸展 → 多部位旋转 → 单腿站立 → 深蹲
```

### 首要层级动作详解

| 序号 | 动作 | 评估内容 | 对应发展姿势 |
|------|------|----------|-------------|
| 1 | **颈椎运动** | 头颈部运动控制与灵活性（屈伸、旋转、侧屈） | 有支撑 |
| 2 | **肩部模式** | 肩带整体功能（外展-外旋-后伸组合模式） | 有支撑/站立 |
| 3 | **多部位屈曲或伸展** | 脊柱分节屈曲与伸展（前屈摸脚尖、后伸） | 站立 |
| 4 | **多部位旋转** | 躯干在站立位的整体旋转能力 | 站立 |
| 5 | **单腿站立** | 单侧平衡与稳定性 | 站立 |
| 6 | **深蹲** | 全身整合功能（含上肢过头要求） | 站立 |

## SFMA 分解（Breakout）算法

### 分解流程总览

```
SFMA首要层级评估
    │
    ├── 标记每个模式：FN / FP / DN / DP
    │
    ├── FP或DP模式？
    │   └── 是 → 需要医疗评估疼痛来源；暂不纠正这些模式
    │
    ├── 识别所有DN模式
    │   ├── DN ≤ 3个 → 逐一处理
    │   └── DN > 3个 → 仅处理1-2个最基本的模式
    │
    ├── 选择优先级最高的DN模式
    │   （颈椎 → 肩部 → 多部位屈伸 → 多部位旋转 → 单腿站立 → 深蹲）
    │
    ├── 分解（Breakout）：
    │   ├── 问题来源于软组织/关节自由度丧失？
    │   │   └── 进行被动活动度测试、关节活动度评估
    │   └── 或来源于完整性/稳定性/运动控制丧失？
    │       └── 进行运动控制测试、姿势稳定性评估
    │
    ├── 应用针对性干预
    │   ├── 灵活性干预（手法/软组织/主动辅助/动态拉伸）
    │   └── 稳定性干预（辅助/主动/RNT）
    │
    └── 重新测试：DN → FN？模式改善？
        ├── 是 → 确认该部分是问题驱动因素；进阶到下一个
        └── 否 → 排除该部分；选择其他DN模式
```

### 分解逻辑的核心原则

**为什么要从最简单的模式开始？**

更复杂的模式（如多部位屈曲）包含了简单模式（如颈部和肩部）的组成部分。简单模式的功能障碍实际上可能是导致复杂模式不良动作的根本原因。

**示例：前屈功能障碍的处理路径**
```
患者：多部位屈曲DN + 颈椎功能障碍
    │
    ├── 花3分钟处理颈椎问题
    │   ├── 屈曲DN → FN？→ 证明颈部是薄弱环节 ✓
    │   ├── 动作质量有所提高？→ 表明颈部是问题的一部分
    │   └── 没有任何变化？→ 颈部功能障碍与多部位屈曲无关
    │
    └── 如颈部非驱动因素，继续向上游分解（肩部→胸椎→髋）
```

### 分解方向的具体路径

每个首要层级模式的分解会指向以下三类根源之一：
1. **软组织伸展性缺陷**（Tissue Extensibility Dysfunction, TED）— 肌肉/筋膜长度不足
2. **关节灵活性缺陷**（Joint Mobility Dysfunction, JMD）— 关节囊/关节面活动受限
3. **运动控制缺陷**（Motor Control Dysfunction, MCD）— 稳定性/时序/协调问题

## SFMA 治疗层级结构

SFMA的纠正优先级遵循与FMS相同的从简单到复杂的逻辑：

```
颈椎 → 肩部 → 多部位屈曲/伸展 → 多部位旋转 → 单腿站立 → 深蹲
```

### 姿势层级与SFMA分解的对应

| 姿势 | SFMA评估/分解动作 |
|------|-------------------|
| **站立** | 深蹲、单腿站立、多部位屈伸、多部位旋转 |
| **堆叠** | 分解动作：长坐躯干旋转、窄距单膝跪地 |
| **悬空** | 分解动作：四足支撑摇摆、腰椎锁定旋转 |
| **有支撑** | 滚动、肩部灵活性、头颈部运动 |

> "\*在站立位筛查这些动作，但在基本姿势（有支撑）下评估和纠正通常会揭示完整的功能障碍情况。"

## SFMA 应用关键指引

### 不适合进行SFMA的情况
- 急性损伤或术后（大量炎症反应——患者未准备好，需要多次就诊才能就绪）
- 患者明显无法进行基本功能性活动
- 首要任务不是SFMA，而是确保呼吸正常、睡眠充足、补足水分和能量

### 临床决策优先事项
1. 首要任务：让患者摆脱疼痛
2. 寻找DN模式（无痛的功能障碍）——这是最安全的干预目标
3. 处理DN模式后重新测试疼痛模式——间接改善可能发生
4. 家庭练习：从最小有效剂量开始，2-3个练习即可
5. 48小时后的复查：变化是否保持？变化保持→全速前进；未保持→重新检查

### 诊断概念
> "医学诊断并没有告诉我们该做什么，它告诉我们不要做什么——要避免的姿势、负荷和感觉。通过SFMA得出的功能诊断结果告诉我们可能需要避免哪些动作，以及哪些动作可以从纠正方法中受益。"
> 
> "我们治疗的不是诊断结果，我们治疗的是患者。"

## SFMA 与 FMS 的协同使用

| 场景 | 使用工具 |
|------|----------|
| 健康人群初次评估 | FMS → 建立功能基线 |
| FMS中发现疼痛（0分） | → 转介 SFMA 评估 |
| 临床诊断不明确 | SFMA 首要层级 → 分解 |
| 出院/康复完成 | FMS 出院筛查 → 确保功能恢复至可接受水平 |
| 损伤后重返运动 | FMS + FCS 组合筛查 |

**标准化交接语言：** 当临床医生使用SFMA标记DN模式后，可以将DN模式与纠正策略传递给教练，让教练在训练环节中继续强化。双方使用相同的DN/FN/DP/FP语言，避免了"该听谁的"拉锯战。

> "改善其他部位的功能障碍会影响疼痛吗？在解决了一个无疼痛的功能障碍模式后，重新测试疼痛模式，查看动作质量或疼痛是否有所改善。"
